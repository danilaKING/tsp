# AI Mock Interviewer — Симулятор технических собеседований

##  Проработка концепции

- **Пользователи:** Junior или разработчик уровня Middle после долгого перерыва.
    
- **Нынешние неудобства:** Дорого, стрессово и неэффективно.
    
- **Сегменты аудитории:**
    
    - **«Паникующий Джун»:** Знает теорию, но теряется при вопросах.
        
    - **«Свитчер»:** Переходит на другой стек. Нужно понять специфические вопросы нового стека.
        
    - **«Забывчивый Сеньор»:** Давно работает в одной компании, забыл алгоритмы и базу.


---

##  Валидация и MVP

### 1. Ключевые гипотезы

1. Пользователи готовы тренироваться с ботом, потому что это снимает социальное напряжение.
    
2. Пользователю важнее получить развернутый разбор ошибок после интервью, чем сам факт диалога.
    
3. Для MVP достаточно текстового чата, голосовой ввод — дополнительно.
    

### 2. Моделирование интервью и его обработка

**Результат опроса:** Главная ценность нашего приложения — **детальный Feedback Report** (отчет с разбором ошибок и ссылками на документацию).

### 3. Определение MVP (Minimum Viable Product)

1. **Настройка:** Пользователь заходит на лендинг (без регистрации или простая форма), выбирает: стек (Python Backend), уровень (Junior), тему (Базы данных / Алгоритмы).
    
2. **Процесс:** Открывается чат. AI-интервьюер задает вопрос. Пользователь пишет ответ текстом. AI реагирует (хвалит, просит уточнить или задает следующий вопрос). Цикл из 5 вопросов.
    
3. **Финал:** Система генерирует «Карту компетенций» и ставит оценку, выделяя положительные моменты и ошибки. Предлагает ссылки на материалы для изучения.
    

> Мы делаем **инструмент для получения обратной связи**, которой так не хватает джунам при подготовке к собеседованиям.

##  Технологии и Архитектура

### Компоненты и Технологии

|**Компонент**|**Технология**|
|---|---|
|Frontend|React + TypeScript|
|Backend|Python + FastAPI|
|База данных|PostgreSQL + SQLAlchemy|
|Интеграция LLM|LlamaIndex/LangChain|
|LLM Provider|OpenAI API|

### Роли LLM

- **Роль 1: «Интервьюер» (В реальном времени)**
    
    - _Задача:_ Вести диалог, задавать технические вопросы по выбранному стеку, реагировать на ответы пользователя, не выходя из роли.
        
- **Роль 2: «Аналитик-Ментор» (Пост-обработка)**
    
    - _Задача:_ Когда собеседование окончено, бэкенд берет весь транскрипт диалога и отправляет его в LLM.
        

### Схема архитектуры

- **Client Layer:** React UI (Frontend). Взаимодействие через REST (config/start) и WebSocket/HTTP (chat).
    
- **Server Layer:** FastAPI (REST + WS) <-> LlamaIndex Context Builder <-> LLM API (prompt / text/JSON report).
    
- **Data Layer:** PostgreSQL (Sessions / Users / Messages).

## Технические требования

### 2.1. Требования к интерфейсу

- **Тип приложения:** Одностраничное веб-приложение.
    
- **Структура:**
    
    1. **Главный экран (Landing/Setup):** Минималистичный дизайн. Форма выбора параметров: Выпадающий список «Стек», переключатель «Уровень», кнопка «Начать собеседование».
        
    2. **Экран собеседования (Chat UI):** Основная область: Лента сообщений. Сообщения от AI выравниваются по левому краю, от пользователя — по правому. Индикация набора текста («AI печатает...»), поле ввода текста с кнопкой отправки. Кнопка принудительного завершения интервью.
        
    3. **Экран результатов (Dashboard):** Визуализация общей оценки (круговая диаграмма или прогресс-бар 0-100%).
        

### 2.2. Требования к технологиям

- **Frontend:** TypeScript, React (18+), Tailwind CSS, Vite.
    
- **Backend:** Python 3.10+, FastAPI.
    
- **Взаимодействие с AI:** LangChain или прямой доступ к OpenAI API / Anthropic API.
    
- **СУБД:** PostgreSQL (хранение профилей, истории чатов, справочников вопросов и результатов). ORM: SQLAlchemy.
    
- **Инфраструктура:** Docker и Docker Compose.
    
- **Система контроля версий:** Git.

## Описание ролей и модулей

### Роли пользователей

- **Пользователь (Кандидат):** Основной потребитель сервиса. Имеет доступ к функционалу прохождения интервью и просмотра собственной статистики.
    
- **Администратор:** Техническая роль. Имеет доступ к управлению системными промптами и просмотру общей статистики.
    

### Перечень основных модулей

- **Модуль 1: Аутентификация и Профиль (User & Admin):** Регистрация, авторизация, управление профилем.
    
- **Модуль 2: Конфигурация и проведение интервью (User):**
    
    - Настройка сессии (стек, уровень, язык).
        
    - Чат-процесс (Real-time ответы, завершение интервью).
        
- **Модуль 3: Аналитика и отчетность (User):**
    
    - Генерация отчета после чата.
        
    - Просмотр Score (0-100), списка ошибок и рекомендаций.
        
- **Модуль 4: Управление контентом (Admin):**
    
    - Редактирование системных инструкций LLM.
        
    - Просмотр логов диалогов для отладки галлюцинаций.

## Описание сущностей БД и данных

### Сущности БД

- **Users:** id (UUID), email, password_hash, created_at.
    
- **Interviews:** id (UUID), user_id, topic (стек), difficulty, status (active/completed/aborted), started_at / finished_at.
    
- **Messages:** id (UUID), interview_id, role (user/assistant), content, timestamp.
    
- **Feedbacks:** id (UUID), interview_id, score (0-100), analysis_json (плюсы, минусы, рекомендации).
    

### Форматы данных

- **Входные (Client -> Server):** JSON (session_id, text).
    
- **Выходные (Server -> Client):** SSE или WebSocket для чата; JSON-объект для отчета.
    
- **Хранение:** PostgreSQL 14+. Ежедневный бэкап в облако (S3/Google Drive), хранение за последние 7 дней.

## Производительность и Безопасность

### Производительность

- Поддержка до 50 одновременных пользователей.
    
- Открытие страниц < 1 секунды.
    
- Начало генерации ответа бота: 3-5 секунд.
    
- Полная генерация ответа < 30 секунд.
    
- Визуализация процесса печати.
    

### Безопасность

- Хеширование паролей.
    
- API-ключи LLM хранятся в переменных окружения (не на клиенте).
    
- Admin API доступны только пользователям с ролью ADMIN.
    
- Использование HTTPS в продакшн-режиме.

## Схемы Use Case (Система симуляции интервью)

1. **Новый пользователь:** Создание учетной записи -> (include) Сохранение профиля, Указание базовых предпочтений, Ввод регистрационных данных.
    
2. **Авторизованный пользователь (Интервью):** Выбор параметров -> Прохождение симуляции -> (include) Диалог с AI (Сохранение истории, Управление состоянием, Обработка ответов).
    
3. **Авторизованный пользователь (Отчет):** Запрос отчета -> (include) Агрегация данных -> Передача в аналитику (LLM) -> Генерация структурированного отчета -> Отображение отчета.
